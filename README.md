This software uses standard security practices. It isolates your web projects from the rest of your computer files. This sandbox design keeps your sensitive documents safe. If you need help, check your log files. These files contain details about recent actions. You can find them in the logs folder within the installation directory. Please keep your software updated to get the latest fixes. The system alerts you when a new version appears on the website.

## 📋 Best practices for daily use

Use folders with simple names. Avoid spaces or special characters in your project folder names. This prevents errors when the server reads your files. Always save your work before you restart the application. If the server does not respond, click the stop button and wait five seconds. Click the start button to restore the connection. You can run multiple projects at the same time by opening new tabs in the interface. Each tab acts as a separate server for your needs.

## 🌐 Connectivity settings

Most users do not need to change the network settings. The default configuration connects to the local loopback address. If you need to access your project from another computer on your network, open the network tab. Select the option for network discovery. Ensure your Windows firewall allows traffic for this application. You might see a pop-up window during the first launch. Please select allow to ensure the server works correctly.

## 📁 Troubleshooting tips

If something stops working, run the diagnostic tool. You can find this tool in the settings menu. It scans your hardware and software for conflicts. It provides a text report on your screen. Copy this report if you need to show it to an expert. Most issues resolve after a simple restart. Never force close the application via the task manager. Use the exit button in the menu to ensure the files close correctly. This habit prevents data corruption in your project folders.
